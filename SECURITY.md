# Меры безопасности SkySEO Backend

## ✅ Реализованные меры защиты

### 1. Защита авторизации

#### Защита от брутфорса

- ✅ Счетчик неудачных попыток входа
- ✅ Автоматическая блокировка после 5 неудачных попыток
- ✅ Сохранение времени последней неудачной попытки
- ✅ Сброс счетчика при успешном входе

#### Защита паролей

- ✅ Хеширование bcrypt (10 раундов)
- ✅ Минимальная длина пароля 6 символов
- ✅ Пароли никогда не возвращаются в API

#### JWT токены

- ✅ Срок действия 7 дней
- ✅ Подпись секретным ключом
- ✅ Проверка активности пользователя при каждом запросе

### 2. Защита регистрации

#### Защита от одноразовых email

- ✅ Блокировка популярных одноразовых доменов:
  - tempmail.com
  - guerrillamail.com
  - 10minutemail.com
  - throwaway.email
  - mailinator.com

#### Защита от множественных регистраций

- ✅ Максимум 3 регистрации с одного IP за час
- ✅ Сохранение IP регистрации
- ✅ Сохранение IP последнего входа

### 3. Защита от накрутки баллов

#### Валидация выполнения задач

- ✅ Минимальная длительность: 30 секунд
- ✅ Максимальная длительность: 10 минут
- ✅ Количество страниц: 2-10
- ✅ Позиция в выдаче: 1-50
- ✅ Обязательная позиция если сайт найден

#### Лимиты на создание задач

- ✅ Максимум 50 задач за 5 минут
- ✅ Проверка баланса перед созданием
- ✅ Автоматическое списание баллов

#### Лимиты на выполнение задач

- ✅ Максимум 100 выполнений за 5 минут
- ✅ Блокировка при подозрительно быстрых выполнениях (< 10 сек)
- ✅ Максимум 20 быстрых выполнений за час

### 4. Rate Limiting

#### Общие лимиты

- ✅ 100 запросов в минуту на пользователя/IP
- ✅ Автоматическая блокировка при превышении
- ✅ HTTP 429 Too Many Requests

### 5. Защита данных

#### Валидация входных данных

- ✅ class-validator для всех DTO
- ✅ Проверка типов данных
- ✅ Whitelist (удаление лишних полей)
- ✅ Transform (преобразование типов)

#### Защита от SQL инъекций

- ✅ Prisma ORM (параметризованные запросы)
- ✅ Нет прямых SQL запросов

#### CORS

- ✅ Настроен для конкретного frontend URL
- ✅ Credentials: true

### 6. Мониторинг подозрительной активности

#### Флаги пользователей

- ✅ `isSuspicious` - флаг подозрительной активности
- ✅ `isActive` - возможность блокировки

#### Сохранение метаданных

## ⚠️ Рекомендации для production

### Критичные

1. **HTTPS обязателен**

   ```nginx
   server {
     listen 443 ssl;
     ssl_certificate /path/to/cert.pem;
     ssl_certificate_key /path/to/key.pem;
   }
   ```

2. **Сильный JWT_SECRET**

   ```bash
   # Генерация случайного ключа
   openssl rand -base64 64
   ```

3. **Переменные окружения**
   - Никогда не коммитить .env
   - Использовать secrets manager (AWS Secrets, Vault)

4. **Обновление зависимостей**
   ```bash
   npm audit fix
   npm update
   ```

### Важные

5. **Helmet.js для HTTP заголовков**

   ```bash
   npm install --save helmet
   ```

   ```typescript
   import helmet from 'helmet'
   app.use(helmet())
   ```

6. **Логирование**

   ```bash
   npm install --save winston
   ```

   - Логировать все подозрительные действия
   - Логировать ошибки
   - Ротация логов

7. **Мониторинг**
   - Sentry для ошибок
   - Prometheus для метрик
   - Grafana для визуализации

8. **Резервное копирование БД**
   ```bash
   # Ежедневный бэкап
   pg_dump skyseo > backup_$(date +%Y%m%d).sql
   ```

### Дополнительные

9. **Rate limiting на уровне Nginx**

   ```nginx
   limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
   limit_req zone=api burst=20;
   ```

10. **Firewall правила**
    - Закрыть все порты кроме 80, 443
    - Whitelist для БД (только backend)

11. **Двухфакторная аутентификация**
    - Для админов обязательно
    - Для пользователей опционально

12. **Webhook подпись**
    ```typescript
    // Проверка подписи YooKassa
    const signature = crypto
    	.createHmac('sha256', SECRET)
    	.update(JSON.stringify(body))
    	.digest('hex')
    ```

## 🔍 Мониторинг подозрительной активности

### Что отслеживать

1. **Множественные регистрации**
   - С одного IP
   - С похожими email
   - В короткий промежуток времени

2. **Подозрительные выполнения**
   - Слишком быстрые (< 30 сек)
   - Слишком много за короткое время
   - Всегда одинаковые результаты

3. **Аномальный баланс**
   - Резкий рост баланса
   - Много задач без выполнений
   - Много выполнений без задач

4. **Подозрительные платежи**
   - Много мелких платежей
   - Платежи сразу после регистрации
   - Возвраты платежей

### Автоматические действия

```typescript
// Пример: автоматическая блокировка
if (suspiciousActivity) {
	await usersService.markAsSuspicious(userId)
	await telegramService.sendAdminNotification(
		`⚠️ Подозрительная активность: ${userId}`,
	)
}
```

## 🚨 Что делать при взломе

1. **Немедленно**
   - Остановить сервер
   - Изменить все пароли
   - Изменить JWT_SECRET
   - Проверить логи

2. **Анализ**
   - Найти точку входа
   - Оценить ущерб
   - Проверить БД на изменения

3. **Восстановление**
   - Восстановить из бэкапа
   - Закрыть уязвимость
   - Обновить зависимости
   - Запустить сервер

4. **Уведомления**
   - Уведомить пользователей
   - Сбросить все токены
   - Потребовать смену паролей
