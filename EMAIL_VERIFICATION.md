# Система подтверждения Email

## Обзор

Реализована полная система подтверждения email адресов для пользователей SkySEO.

## Функционал

### 1. При регистрации

- Генерируется уникальный токен подтверждения (`emailVerificationToken`)
- Отправляется 2 письма:
  - Приветственное письмо с упоминанием о необходимости подтверждения
  - Письмо с ссылкой для подтверждения email
- Пользователь может войти в систему, но не может пополнять баланс

### 2. Подтверждение email

- Ссылка ведет на `https://skyseo.site/verify-email?token=xxx`
- При переходе по ссылке:
  - Токен проверяется в базе данных
  - Устанавливается `emailVerified = true`
  - Токен удаляется (`emailVerificationToken = null`)
  - Показывается сообщение об успехе

### 3. Ограничения без подтверждения

- **Пополнение баланса заблокировано** - PaymentsService проверяет `emailVerified`
- В приложении показывается желтый баннер с предупреждением
- Кнопка "отправить повторно" для повторной отправки письма

### 4. В приложении

- Баннер с предупреждением если email не подтвержден
- Кнопка повторной отправки письма подтверждения
- Информация о том, что нужно проверить папку "Спам"

## API Endpoints

### Backend (`/auth`)

- `POST /auth/verify-email` - подтверждение email по токену
- `POST /auth/resend-verification` - повторная отправка письма (требует авторизации)

### Frontend страницы

- `/verify-email?token=xxx` - страница подтверждения email
- `/forgot-password` - восстановление пароля (обновлена в стиле сайта)
- `/reset-password?token=xxx` - сброс пароля (обновлена в стиле сайта)

## База данных

### Новые поля в таблице `users`:

```sql

emailVerificationToken VARCHAR UNIQUE
```

```bash

```

## Email шаблоны

### 1. Приветственное письмо

- Упоминание о 1000 баллов бонуса
- Информация о необходимости подтверждения email
- Напоминание проверить папку "Спам"

### 2. Письмо подтверждения

- Кнопка с ссылкой на подтверждение
- Предупреждение о невозможности пополнения без подтверждения
- Напоминание о папке "Спам"

### 3. Все письма отправляются с `info@skyseo.site` через Resend

## Безопасность

### Токены подтверждения:

- Генерируются с помощью `crypto.randomBytes(32).toString('hex')`
- Уникальные в базе данных
- Одноразовые (удаляются после использования)
- Не имеют срока истечения (в отличие от токенов сброса пароля)

### Логирование:

- Логируются только домены email (не полные адреса)
- Логируются частичные ID пользователей (первые 8 символов)
- НЕ логируются токены и пароли

## Тестирование

### Сценарий 1: Новая регистрация

1. Зарегистрироваться в приложении
2. Проверить получение 2 писем
3. Войти в приложение - должен показаться баннер
4. Попробовать пополнить баланс - должна быть ошибка
5. Перейти по ссылке подтверждения
6. Проверить что баннер исчез
7. Попробовать пополнить баланс - должно работать

### Сценарий 2: Повторная отправка

1. В приложении нажать "отправить повторно"
2. Проверить получение нового письма
3. Старая ссылка должна перестать работать
4. Новая ссылка должна работать

### Сценарий 3: Уже подтвержденный email

1. Перейти по ссылке подтверждения повторно
2. Должно показаться "Email уже подтвержден"
