## 🚀 Как запустить?

### Вариант 1: Через скрипт (рекомендуется)

```bash
cd skyseo_back
./seed-improved.sh
```

### Вариант 2: Напрямую

```bash
cd skyseo_back
npx ts-node --transpile-only prisma/seed-improved.ts
```

### Вариант 3: Через npm

```bash
cd skyseo_back
npm run seed  # Запустит старый сидер
# или
npx ts-node --transpile-only prisma/seed-improved.ts  # Новый сидер
```

## ⚠️ Важно!

1. **Перед запуском** убедитесь, что база данных запущена:

   ```bash
   docker-compose up -d
   ```

2. **Если нужно очистить базу** перед сидом:

   ```bash
   npx prisma migrate reset
   ```

   Это удалит все данные и применит миграции заново.

3. **Время выполнения**: ~2-5 минут (генерируется много данных)

## 📱 Что проверить после запуска?

1. Войдите в приложение под **asd11@gmail.com** / **password123**
2. Откройте любой из 3 сайтов
3. Перейдите на вкладку **"Обзор"**:
   - Должны быть заполнены все метрики
   - Видимость в топ-10 должна быть > 0%
   - Средние позиции должны быть в диапазоне 5-20
4. Перейдите на вкладку **"Ключевые слова"**:
   - Должны быть все ключевики с позициями
   - Колонки "Яндекс" и "Google" заполнены
   - Есть значения ±7д (положительные = улучшение)
   - Есть количество визитов

## 🐛 Проблемы?

Если что-то не работает:

1. Проверьте подключение к базе данных
2. Убедитесь, что миграции применены: `npx prisma migrate dev`
3. Проверьте логи в консоли при запуске сидера
4. Попробуйте очистить базу: `npx prisma migrate reset`
